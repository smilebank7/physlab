# Code of Conduct

physlab follows the spirit and structure of Contributor Covenant v2.1. The goal
is a small project space where robotics, simulation, and tooling work can be
discussed directly without becoming hostile.

## Our Standards

Examples of behavior that helps the project:

- Being respectful of differing experience levels and constraints.
- Giving actionable technical feedback.
- Accepting responsibility for mistakes and correcting them.
- Keeping discussions focused on reproducible behavior, evidence, and scope.

Examples of unacceptable behavior:

- Harassment, insults, threats, or sustained disruption.
- Sexualized language or imagery in project spaces.
- Publishing private information without explicit permission.
- Dismissing safety, license, or scope concerns in bad faith.

## Scope

This code applies in repository issues, pull requests, discussions, release
channels, and other project-managed spaces. It also applies when someone
represents the project in public.

## Enforcement

Maintainers may remove comments, close threads, reject contributions, or block
participants when behavior harms the project space. Enforcement should be
proportional, documented when practical, and focused on restoring a workable
collaboration environment.

Reports can be sent privately to the project owner or raised through GitHub
moderation tools. Maintainers will avoid sharing report details beyond what is
needed to respond.
